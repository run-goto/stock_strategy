# 《短线操盘实战技巧》策略实现说明

本文档说明基于《短线操盘实战技巧》一书实现的6个核心量化策略。

## 策略列表

### ST-01: 双均线趋势策略 (DualMATrendStrategy)

**策略逻辑**: 基于日线收盘价与MA100/MA60的关系判断趋势。

**入场条件**:
- 当日收盘价 > MA100（首次上穿）
- 当日成交量 > 20日均量

**出场条件**:
- 收盘价 < MA60（首次下穿）
- 或从最高点回撤超过8%

**止损设置**: 入场当日最低价下方1%

**适用场景**: 趋势跟踪，适合中长期持有

---

### ST-02: 低位123结构突破策略 (Low123BreakoutStrategy)

**策略逻辑**: 识别日线级别的低位123结构，突破高点2时入场。

**形态识别规则**:
1. 找到阶段性低点1（20日最低）
2. 低点1之后出现反弹高点2（反弹幅度 > 5%）
3. 随后回落形成低点3，要求：低点3 > 低点1，且低点3出现后10日内未创新低

**入场条件**: 收盘价突破高点2的价格水平（突破幅度 ≥ 1%）

**止损设置**: 初始止损设于低点3下方1%

**适用场景**: 底部反转形态，捕捉趋势启动点

---

### ST-03: MACD底背离双突破策略 (MACDDivergenceBreakoutStrategy)

**策略逻辑**: 股价创新低而MACD黄白线（DIFF）未创新低，随后价格突破趋势线与水平阻力线时入场。

**底背离识别**:
1. 寻找60日内的两个显著低点（波谷）
2. 股价低点B < 低点A，且对应的DIFF值低点b ≥ 低点a（允许误差 ±3%）

**双突破条件**:
1. 绘制连接低点A之前的反弹高点与低点B之后近期高点的下降趋势线
2. 以低点B之后的反弹高点做水平阻力线
3. 当日收盘价同时站上趋势线延伸值和水平阻力线

**止损设置**: 初始止损设于低点B下方2%

**适用场景**: 超跌反弹，技术面背离信号

---

### ST-04: 向上突破缺口追涨策略 (GapBreakoutStrategy)

**策略逻辑**: 股价以跳空缺口方式突破关键阻力位，视为强势信号，当日追涨。

**缺口识别**: 当日最低价 > 前一日最高价，形成向上跳空缺口

**关键阻力定义**:
- 前60日最高价
- 前期震荡区间上沿
- 或MA100均线

**入场条件**:
1. 缺口突破任一关键阻力位
2. 缺口幅度 ≥ 1%
3. 当日成交量 > 20日均量的1.5倍

**止损设置**: 缺口上沿价格下方0.5%

**适用场景**: 强势突破，快速上涨行情

---

### ST-05: 缺口回踩支撑买入策略 (GapPullbackStrategy)

**策略逻辑**: 出现突破缺口后，等待价格自然回踩缺口上沿，确认支撑有效后入场。

**前置条件**: 20个交易日内曾出现符合策略ST-04定义的突破缺口

**回踩确认条件**:
1. 股价从缺口形成后的高点回落，最低价触及缺口上沿 ±1% 区间
2. 回落过程中成交量萎缩（< 20日均量）
3. 随后出现放量阳线（涨幅 > 2% 且成交量 > 20日均量）

**入场信号**: 确认阳线的收盘价

**止损设置**: 缺口上沿下方1%

**适用场景**: 突破后的二次买点，风险相对较低

---

### ST-06: 强势涨停突破策略 (StrongLimitUpBreakoutStrategy)

**策略逻辑**: 涨停板当日突破前期关键高点，视为强势启动信号，次日择机介入。

**涨停识别**:
- 当日涨幅 ≥ 9.8%（考虑四舍五入）
- 收盘价为当日最高价

**突破条件**: 涨停当日最高价突破前60日最高价

**过滤条件**:
1. 流通市值 < 100亿（剔除大盘股）
2. 换手率介于3%～20%之间

**入场时机**: 次日开盘价（若高开超过5%则放弃）

**止损设置**: 涨停阳线实体的中位价格

**适用场景**: 强势股追涨，短线爆发力强

## 使用方式

### 1. 启用策略

在 `backend/config/app_config.yaml` 中配置策略开关：

```yaml
strategies:
  DualMATrendStrategy:
    enabled: true
  Low123BreakoutStrategy:
    enabled: true
  MACDDivergenceBreakoutStrategy:
    enabled: true
  GapBreakoutStrategy:
    enabled: true
  GapPullbackStrategy:
    enabled: true
  StrongLimitUpBreakoutStrategy:
    enabled: true
```

### 2. 运行扫描

```bash
# 启动API服务
python main.py

# 创建扫描任务
curl -X POST http://127.0.0.1:8000/api/v1/scans \
  -H "Content-Type: application/json" \
  -d '{"start": "20260101", "end": "20260412", "targets": ["20260412"]}'

# 查询任务结果
curl http://127.0.0.1:8000/api/v1/scans/{job_id}/results
```

### 3. 测试策略

```bash
# 运行单元测试
python tests/test_short_term_strategies.py
```

## 注意事项

1. **数据要求**: 不同策略需要不同的历史数据长度
   - ST-01: 至少101天（计算MA100）
   - ST-02: 至少60天
   - ST-03: 至少80天
   - ST-04: 至少61天
   - ST-05: 至少40天
   - ST-06: 至少61天

2. **风险控制**: 
   - 所有策略都应设置止损
   - 建议结合过滤器FL-02（大盘环境判定器）使用
   - 弱势市场建议降低仓位或空仓

3. **参数调优**: 
   - 当前参数为默认值，可根据实际回测结果调整
   - 建议使用Walk-Forward分析验证参数稳定性

4. **未来函数防范**: 
   - 回测时使用次日开盘价作为成交价
   - 避免使用当日收盘价导致结果虚高

## 策略组合建议

| 市场环境 | 推荐策略组合 |
|---------|------------|
| 强势市场 | ST-01 + ST-04 + ST-06 |
| 震荡市场 | ST-02 + ST-05 |
| 弱势市场 | ST-03（轻仓）或空仓观望 |
