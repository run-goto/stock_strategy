## 1. 项目概述

### 1.1 项目背景
基于《短线操盘实战技巧》一书提炼的11个专业能力模块，需将其中可量化的部分开发为自动化交易系统，实现策略信号生成、选股过滤、风险控制的一体化运行。

### 1.2 产品目标
- **策略引擎**：实现6个可编程交易策略的实时信号生成与回测。
- **选股过滤器**：实现3个量化选股模型，每日盘后/盘中输出符合条件的股票池。
- **集成框架**：策略与选股器通过统一数据接口运行，支持历史回测与模拟/实盘交易。

### 1.3 用户角色
- **量化研究员**：使用回测功能验证策略有效性。
- **交易员**：接收实时信号，进行人工或半自动下单。
- **系统管理员**：维护数据源、监控系统运行状态。

## 2. 功能需求

### 2.1 策略引擎模块（6个量化策略）

#### 策略 ST-01：双均线趋势策略

| 需求项 | 详细说明 |
| :--- | :--- |
| **策略逻辑** | 基于日线收盘价与MA100/MA60的关系判断趋势。 |
| **入场信号** | 当日收盘价 > MA100（首次上穿），且当日成交量 > 20日均量。 |
| **出场信号** | 收盘价 < MA60（首次下穿） 或 从最高点回撤超过8%。 |
| **初始止损** | 入场当日最低价下方1%。 |
| **仓位管理** | 初始仓位为总资金的20%，趋势确认后（MA20上穿MA60）加仓至40%。 |
| **适用品种** | 全A股（剔除ST、上市不满60日新股） |
| **输出内容** | 信号类型（BUY/SELL/ADD）、信号价格、建议仓位、止损价 |

#### 策略 ST-02：低位123结构突破策略

| 需求项 | 详细说明 |
| :--- | :--- |
| **策略逻辑** | 识别日线级别的低位123结构，突破高点2时入场。 |
| **形态识别规则** | 1. 找到阶段性低点1（20日最低）。<br>2. 低点1之后出现反弹高点2（反弹幅度 > 5%）。<br>3. 随后回落形成低点3，要求：低点3 > 低点1，且低点3出现后10日内未创新低。 |
| **入场信号** | 收盘价突破高点2的价格水平（突破幅度 ≥ 1%）。 |
| **止损设置** | 初始止损设于低点3下方1%。 |
| **出场信号** | 价格跌破最近10日低点 或 盈利达到15%后出现5%回撤。 |
| **输出内容** | 信号类型（BUY）、入场价、止损价、结构图示（高点2、低点1/3坐标） |

#### 策略 ST-03：MACD底背离双突破策略

| 需求项 | 详细说明 |
| :--- | :--- |
| **策略逻辑** | 股价创新低而MACD黄白线（DIFF）未创新低，随后价格突破趋势线与水平阻力线时入场。 |
| **底背离识别** | 1. 寻找60日内的两个显著低点（波谷）。<br>2. 股价低点B < 低点A，且对应的DIFF值低点b ≥ 低点a（允许误差 ±3%）。 |
| **双突破条件** | 1. 绘制连接低点A之前的反弹高点与低点B之后近期高点的下降趋势线。<br>2. 以低点B之后的反弹高点做水平阻力线。<br>3. 当日收盘价同时站上趋势线延伸值和水平阻力线。 |
| **止损设置** | 初始止损设于低点B下方2%。 |
| **出场信号** | 价格跌破MA20 或 MACD出现死叉。 |
| **特殊处理** | 若出现二次底背离，需标记为“多重背离”，建议轻仓。 |
| **输出内容** | 背离状态（首次/二次）、突破有效性确认、建议入场区间 |

#### 策略 ST-04：向上突破缺口追涨策略

| 需求项 | 详细说明 |
| :--- | :--- |
| **策略逻辑** | 股价以跳空缺口方式突破关键阻力位，视为强势信号，当日追涨。 |
| **缺口识别** | 当日最低价 > 前一日最高价，形成向上跳空缺口。 |
| **关键阻力定义** | 前60日最高价、前期震荡区间上沿、或MA100均线。 |
| **入场条件** | 1. 缺口突破任一关键阻力位。<br>2. 缺口幅度 ≥ 1%。<br>3. 当日成交量 > 20日均量的1.5倍。 |
| **入场时机** | 信号触发当日的收盘价（模拟次日开盘价买入）。 |
| **止损设置** | 缺口上沿价格下方0.5%。 |
| **出场信号** | 价格回补缺口（收盘价低于缺口上沿）或 持有超过5个交易日。 |
| **输出内容** | 缺口幅度、突破的阻力类型、建议仓位（轻仓试探） |

#### 策略 ST-05：缺口回踩支撑买入策略

| 需求项 | 详细说明 |
| :--- | :--- |
| **策略逻辑** | 出现突破缺口后，等待价格自然回踩缺口上沿，确认支撑有效后入场。 |
| **前置条件** | 20个交易日内曾出现符合策略ST-04定义的突破缺口。 |
| **回踩确认条件** | 1. 股价从缺口形成后的高点回落，最低价触及缺口上沿 ±1% 区间。<br>2. 回落过程中成交量萎缩（< 20日均量）。<br>3. 随后出现放量阳线（涨幅 > 2% 且成交量 > 20日均量）。 |
| **入场信号** | 确认阳线的收盘价。 |
| **止损设置** | 缺口上沿下方1%。 |
| **出场信号** | 再次跌破缺口上沿 或 创出近期新高后回落3%。 |
| **输出内容** | 原始缺口日期、回踩确认日、支撑有效性评分 |

#### 策略 ST-06：强势涨停突破策略

| 需求项 | 详细说明 |
| :--- | :--- |
| **策略逻辑** | 涨停板当日突破前期关键高点，视为强势启动信号，次日择机介入。 |
| **涨停识别** | 当日涨幅 ≥ 9.8%（考虑四舍五入），且收盘价为当日最高价。 |
| **突破条件** | 涨停当日最高价突破前60日最高价。 |
| **过滤条件** | 1. 涨停时间在上午11:00之前（若有L2逐笔数据）。<br>2. 流通市值 < 100亿（剔除大盘股）。<br>3. 换手率介于3%～20%之间。 |
| **入场时机** | 次日开盘价（若高开超过5%则放弃）。 |
| **止损设置** | 涨停阳线实体的中位价格。 |
| **出场信号** | 跌破5日均线 或 出现涨停后第四日仍未创新高。 |
| **输出内容** | 涨停强度评分（封单占比、突破位置）、建议入场区间、风险提示 |

### 2.2 选股过滤器模块（3个量化模型）

#### 过滤器 FL-01：强势股初选池

| 需求项 | 详细说明 |
| :--- | :--- |
| **选股逻辑** | 筛选处于上升趋势且近期表现强于大盘的个股。 |
| **筛选条件** | 1. 收盘价位于MA100均线之上。<br>2. 20日涨幅跑赢同期上证指数 ≥ 5%。<br>3. RSI(14) > 55。<br>4. 最近5日平均成交量 > 60日平均成交量。<br>5. 剔除ST、*ST、上市不满90日的新股。 |
| **输出内容** | 股票代码、名称、MA100乖离率、相对大盘强度、近5日资金流向（如有） |
| **运行频率** | 每日收盘后自动运行，输出股票池供人工二次筛选。 |

#### 过滤器 FL-02：大盘环境判定器

| 需求项 | 详细说明 |
| :--- | :--- |
| **判定逻辑** | 基于上证指数（000001）的均线状态判断当前市场牛熊/震荡。 |
| **状态定义** | - **强势市场**：指数收盘价位于MA100之上，且MA20 > MA60。<br>- **弱势市场**：指数收盘价位于MA100之下。<br>- **震荡市场**：指数位于MA100上下3%区间内波动超过10个交易日。 |
| **输出内容** | 当前市场状态、建议仓位上限（强势80%、震荡50%、弱势20%或空仓）、风险提示语。 |
| **应用方式** | 作为其他策略的总开关。当状态为“弱势”时，策略引擎仅允许发出卖出信号，不产生买入信号。 |

#### 过滤器 FL-03：热点题材龙头识别器

| 需求项 | 详细说明 |
| :--- | :--- |
| **选股逻辑** | 每日扫描涨停个股，结合板块热度，自动识别潜在龙头。 |
| **数据依赖** | 需要外部数据源提供：个股所属概念板块、板块内个股涨幅排名。 |
| **龙头评分模型** | **评分维度**：<br>- 涨停时间（越早分数越高）<br>- 流通市值（越小分数越高）<br>- 板块内涨停家数占比（越高分数越高）<br>- 连板高度（首板为基础分，连板递增） |
| **输出内容** | 按评分降序排列的候选龙头列表（前10名），包含评分明细、板块热度、建议关注优先级。 |
| **运行频率** | 盘中实时监控（如数据支持）或收盘后运行用于次日计划。 |

## 3. 非功能需求

### 3.1 数据需求

| 数据项 | 用途 | 来源建议 |
| :--- | :--- | :--- |
| 日线行情（开高低收量） | 所有策略的核心计算数据 | AKShare、TuShare、iTick |
| 复权因子 | 准确计算历史回测收益 | 数据源自带 |
| 分钟级行情（可选） | 策略ST-06的涨停时间判断 | 需Level-2数据源 |
| 股票基本信息（市值、行业、上市日期） | 过滤ST、新股、流通市值筛选 | 数据源基础信息接口 |
| 概念板块归属 | 过滤器FL-03的热点识别 | 需额外概念标签数据源 |

### 3.2 性能要求
- 日线策略信号计算（全市场扫描）：< 5分钟。
- 盘中选股器刷新频率：≤ 1分钟（如有实时数据）。
- 历史回测速度：单策略5年日线回测 < 30秒。

### 3.3 扩展性要求
- 策略引擎需采用插件化设计，新增策略只需实现统一接口（`generate_signal(data) -> Signal`）。
- 选股过滤器支持组合条件，用户可自定义AND/OR逻辑。

## 5. 开发优先级与里程碑

| 阶段 | 内容 | 预计交付物 |
| :--- | :--- | :--- |
| **Phase 1 (MVP)** | 实现ST-01、ST-02、ST-04三个基础策略<br>实现FL-01、FL-02两个过滤器<br>完成日线数据接入与本地回测框架 | 可运行的Python回测脚本，输出信号CSV |
| **Phase 2** | 实现ST-03（背离识别）、ST-05（缺口回踩）、ST-06（涨停突破）<br>完成FL-03（龙头识别，需板块数据） | 全策略引擎，支持参数优化 |
| **Phase 3** | 实时行情接入，盘中信号推送<br>可视化看板（信号日历、资金曲线） | Web端监控面板，微信/钉钉推送 |
| **Phase 4** | 对接实盘交易接口（如QMT、PTrade） | 半自动/全自动下单功能 |

## 6. 风险与注意事项

1. **未来函数风险**：回测时必须严格使用“次日开盘价”作为成交价，避免使用当日收盘价导致结果虚高。
2. **涨停板流动性**：ST-06策略需在实盘中考虑涨停封死无法买入的情况，需在PRD中明确：若次日开盘即封死涨停，信号视为无效。
3. **幸存者偏差**：回测数据需包含已退市股票，避免筛选结果过于乐观。
4. **过拟合防范**：所有策略参数需通过Walk-Forward（推进分析）验证，避免单一时间段参数最优。
